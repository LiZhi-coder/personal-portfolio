# BoolCore 测试使用说明

## 📋 测试文件说明

这个测试套件包含以下文件：

1. **`测试文档.md`** - 完整的测试文档和结果记录表格
2. **`test_quick.sh`** - 快速测试脚本（推荐先运行）
3. **`test_functional.sh`** - 详细功能测试脚本  
4. **`test_performance.sh`** - 性能测试脚本
5. **`backend/api_test.go`** - Go单元测试文件

## 🚀 快速开始

### 1. 启动服务
```bash
# 终端1：启动后端
cd backend
go run cmd/server/main.go

# 终端2：启动前端
cd frontend  
npm run dev
```

### 2. 运行快速测试
```bash
# 运行综合测试（推荐）
./test_quick.sh
```

### 3. 运行详细测试
```bash
# 详细功能测试
./test_functional.sh

# 性能测试
./test_performance.sh

# Go单元测试
cd backend && go test -v
```

## 📊 如何填写测试结果

### 方法1：自动记录（推荐）
运行测试脚本后，直接复制终端输出的结果到`测试文档.md`相应表格中。

### 方法2：手动填写
1. 打开`测试文档.md`
2. 找到对应测试用例的表格行
3. 将"_待填入_"替换为实际测试结果
4. 更新状态列：✅通过 / ❌失败

## 🧪 测试用例说明

### 功能测试用例
- **FT-002到FT-005**: 四种输入格式基础测试
- **FT-006**: 边界条件测试（n=1）
- **FT-007**: 大规模函数测试（n=6）
- **FT-008**: 错误处理测试
- **FT-009到FT-011**: 特殊函数类型测试

### 性能测试指标
- **响应时间**: n≤6应<100ms, n≤8应<500ms
- **并发性能**: 50连接应>100req/s
- **内存使用**: 应合理增长，无泄漏

## 🔧 测试环境要求

### 系统要求
- Linux/macOS/Windows
- Go 1.24+
- Node.js 16+
- curl命令
- jq命令（JSON处理）

### 可选工具（性能测试）
```bash
# Ubuntu
sudo apt install wrk apache2-utils

# macOS  
brew install wrk

# 或使用内置的curl进行性能测试
```

## 📈 结果解读

### 成功标准
- ✅ HTTP状态码正确（200/400）
- ✅ 关键字段值符合预期
- ✅ 不同输入格式结果一致
- ✅ 响应时间符合目标
- ✅ 错误处理正确

### 常见问题
- **连接失败**: 检查后端服务是否启动
- **JSON解析错误**: 检查jq是否安装
- **性能差**: 检查系统负载和硬件配置
- **计算结果错误**: 对比SageMath验证

## 🔄 持续集成

如果需要集成到CI/CD，可以参考：

```yaml
# .github/workflows/test.yml 示例
name: BoolCore Tests
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - uses: actions/setup-go@v2
      with:
        go-version: 1.24
    - name: Install dependencies
      run: sudo apt-get install -y jq wrk
    - name: Start backend
      run: cd backend && go run cmd/server/main.go &
    - name: Wait for service
      run: sleep 10
    - name: Run tests
      run: ./test_quick.sh
```

运行这些测试后，你就有了完整的测试数据来填入文档！
